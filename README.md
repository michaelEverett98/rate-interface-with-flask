# rate-interface-with-flask
---

## Minimum Viable Product (MVP):
---

- [ ] Login system with username and password that assigns each user a unique key
- [ ] Create a user interface with HTML, Python, and Flask
- [ ] Interface a songlist which is uneditable by regular users and is linked to an SQL database
- [ ] Ability to input submission name
- [ ] Ability to assign scores and comments to each song
    - [ ] Scores should be automatically corrected to one decimal place
    - [ ] No negative numbers or numbers above 11 should be inputtable
- [ ] Ballot checker button which checks for:
    - [ ] Maximum of one 11 and one 0 given
    - [ ] No scores given between 0-1 and 10-11
    - [ ] Correctly formatted comment i.e. comments don't being with /
- [ ] Ability to save the inputted information to a database
- [ ] Ability to output a .txt file which is automatically formatted correctly to rate standards

## Potential additional features:
---

- [ ] Link the user logins directly to Reddit
- [ ] Contain a link which would send you to the submission page for the rate
- [ ] Have three tiers of user; Admin, Host, Rater
- [ ] Have several rates which a user can cycle between
- [ ] Ability to bring up a ballot visualizer which would contain various stats that may be of interest
- [ ] Ability to import/export csv formats to increase compatibility with other tools the rating community utilises
